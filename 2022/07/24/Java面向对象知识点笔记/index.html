Java面向对象知识点笔记

发表于2022-07-24|更新于2022-08-20
|阅读量:

继承

构造器

  • 子类并不会继承父类的构造器,但是会在构造器内调用父类的构造器
  • 父类的构造器没有参数时会在子类的构造器内默认自动调用
  • 父类的构造器带有参数时,如果需要调用,需要在子类的构造器中显示调用

    重写

  • 重写方法不能抛出新的检查异常或者比被重写方法申明更加宽泛的异常
  • 重写的返回值与形参不可以改变
  • 对象创建时,实例化类可以是声明类的子类,但是调用方法时,声明类必须拥有此方法,否则会报错
  • 子类需要调用父类中被重写的方法时,可以在方法前加super调用 重写规则:
    • 参数列表与被重写的方法参数列表完全相同
    • 返回类型与被重写方法的返回类型可以不相同，但是必须是父类返回值的派生类
    • 访问权限不能比父类中被重写的方法的访问权限更低
    • 父类的成员方法只能被它的子类重写
    • 声明为final的方法不能被重写
    • 声明为static的方法不能被重写，但是能够被再次声明
    • 子类和父类在同一个包中，那么子类可以重写父类所有方法，除了声明为private和final的方法
    • 子类和父类不在同一个包中，那么子类只能够重写父类的声明为public和 protected的非final方法
    • 重写的方法能够抛出任何非强制异常，无论被重写的方法是否抛出异常。但是，重写的方法不能抛出新的强制性异常，或者比被重写方法声明的更广泛的强制性异常，反之则可以。
    • 构造方法不能被重写
    • 如果不能继承一个类，则不能重写该类的方法

    重载

    重载规则:
    • 被重载的方法必须改变参数列表(参数个数或类型不一样)
    • 被重载的方法可以改变返回类型
    • 被重载的方法可以改变访问修饰符
    • 被重载的方法可以声明新的或更广的检查异常
    • 方法能够在同一个类中或者在一个子类中被重载
    • 无法以返回值类型作为重载函数的区分标准
      方法的重写(Overriding)和重载(Overloading)是java多态性的不同表现,可以使程序有良好的扩展，并可以对所有类的对象进行通用处理

    抽象类

  • 抽象类用abstract class来声明
  • 抽象类与普通类没有区别,但是抽象类不可以实例化
  • 但可以通过子类继承抽象类的方法和成员变量

    抽象方法

    • 抽象方法用abstract声明,抽象方法只存在方法名,不存在方法体
    • 一个类包含抽象方法,此类必须为抽象类
    • 子类必须重写父类的抽象方法,或者声明自身为抽象类
    • 构造方法以及用static声明的方法不可以声明为抽象方法

    接口

  • 接口在Java中时一个抽象类型,用interface来声明,一个类需要通过继承接口的方式来继承接口的抽象方法.
  • 接口并不是类 ,他们属于不同的概念
  • 接口中所有的方法必须是抽象方法
  • 接口中没有构造方法
  • 接口不能包含成员变量,除了final和static变量
  • 接口并非被类继承,而是通过implements被类实现
  • 接口可以多继承
  • 接口中的方法会被隐式声明为public abstract(且只能是)
  • 接口可以含有变量,但是会被隐式声明为public static final(且只能是public)
  • 类在实现接口的方法是,不可以抛出强制性异常,只能在接口中,或者继承接口的抽象类中抛出该强制性异常(?没有理解)
  • 类在重写方法时要保持一直的方法名,并且应当保持相同或者兼容的返回值类型
  • 抽象类没必要实现该接口的方法
  • Java中类的多继承不合法,但是接口的多继承是合法的

    标记接口

    • 标记接口不包含任何方法
      标记接口的作用:
    • 建立一个公共的父接口
    • 向一个类添加数据类型

    枚举

  • Java的枚举是 一个特殊的类,一般用来表示一组常量
  • Java的枚举用enum来定义,常量间用,隔开
  • 枚举也可以声明在内部类中
  • 枚举也可以用for语句来迭代枚举元素:
  • 枚举可以像普通类一样用自己的变量,方法和构造函数,构造函数只能用private访问修饰符,所以外部无法调用
  • 枚举可以包含具体方法,也可以包含抽象方法,枚举有抽象方法时,枚举的 每个实例都必须实现它

    包

  • 包的存在是为了防止命名冲突,访问控制,提供搜索和定位类、接口、枚举、和注释等
