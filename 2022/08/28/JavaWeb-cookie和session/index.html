JavaWeb-cookie和session

发表于2022-08-28|更新于2022-08-29
|阅读量:

Cookie的基本使用

  • Cookie:客户端会话技术,将数据保存到客户端,以后每次请求都携带Cookie进行访问
  • Cookie基本使用:

1.创建Cookie对象,使用数据

2.发送Cookie到客户端:使用response对象

3.获取客户端携带的所有Cookie,使用reques对象

4.遍历数组,获取每一个Cookie对象

5.使用Cookie对象方法获取数据

Cookie使用细节

  • Cookie存活时间

    默认情况下,Cookie存储在浏览器内存中,当浏览器关闭,内存释放,则Cookie被销毁

setMaxAge(int seconds):设置Cookie存活时间

1.正数:将Cookie写入浏览器存在的电脑硬盘,持久化储存,到时间自动删除

2.负数:默认值,Cookie存在当前浏览器内存中,当浏览器关闭,则Cookie被销毁

3.0:删除对应Cookie

  • Cooki存储中文

    Tomcat8.5及以上是支持中文Cookie的

    但是8.5以下版本则需要进行转码
    此时会将value转码成URL数据,在获取数据时也是URL数据,需要对数据进行URL解码

    Session

  • 服务端会话技术,将数据保存在服务端
  • Java EE提供HTTLSession接口,来实现一次会话的多次请求之间的数据共享功能

    Session原理

  • Session是基于Cookie实现的

    Session使用细节

    钝化,活化

  • 服务器重启后,Session数据是否还在

    钝化:在服务器正常关闭后,Tomcat会自动将Session数据写入硬盘的文件中

    活化:再次启动服务器后,会从硬盘文件中加载数据到Session中

    销毁

  • 默认情况下,无操作,session会在30分钟后销毁

    可以在web.xml中设置session的保留时间
  • 使用session.invalidate();手动销毁
