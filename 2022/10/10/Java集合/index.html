Java集合

发表于2022-10-10|更新于2022-10-11
|阅读量:

List

基本介绍

  1. List集合类中元素有序(添加顺序和取出顺序一致),且可重复
  2. List集合中的每个元素都有其对应的顺序索引,即支持索引
  3. List集合中的元素都对应一个整数型的序号记载其在容器中的位置,可以根据序号存取容器中的元素.

ArrayList

  1. ArrayList允许加入空值,且允许多个空值
  2. ArrayList是由数组来实现数据存储的
  3. ArrayList基本等于Vector,但是ArrayList是 线程不安全的 ,多线程使用时,不建议使用ArrayList

底层操作机制

  1. ArrayList中维护了一个Object类型的数组elementData
  2. 创建ArrayList对象时,如果使用的是无参构造器,则初始elementData的容量为0,第1次添加,则扩容elementData为10,如需要再次扩容,则扩容elementData为1.5倍.
  3. 如果使用的是指定大小的构造器,则初始容量为指定大小,需要扩容时直接扩容至1.5倍

Vector

  1. Vector底层也是一个对象数组
  2. Vector是线程同步的,即线程安全,Vector类的操作方法都带有synchronized
  3. 在开发中,需要考虑线程同步时,考虑使用Vector

底层操作机制

创建Vector对象时,如果使用的是无参构造器,则初始elementData的容量为10,在容器满了之后按照两倍扩容容器.指定容器大小时,在容器满后会按照当前的两倍扩展容器.

LinkedList

  1. LinkedList底层实现了双向链表和双端队列的特点
  2. 可以添加任意元素(可重复),包括null
  3. 线程不安全,不可同步

Set

基本介绍

  1. 无序(添加和取出顺序不一致),没有索引
  2. 不允许重复元素出现
  3. Set取出数据的顺序是固定的,且不是添加的顺序

HashSet

HashSet实际上是HashMap

底层实现

  1. 添加一个元素时,先得到hash值,转换成索引值
  2. 找到存储数据表table,看这个索引位置是否有已经存放的数据
  3. 如果没有,直接加入
  4. 如果有,则用equals比较,相同即放弃添加,不同则加到最后.
  5. 在Java8中如果一个链表的元素个数到TREEIFY_THRESHOLD(默认是8)且table大小>=MIN_TREEIFY_CAPACITY(默认64)就会进行树化(红黑树)

LinkedHashSet

  1. LinkedHashSet底层是LinkedHashMap,底层维护了一个数组+双向链表的数据结构
  2. LinkedHashSet根据元素的hashCode值来决定元素的储存位置,同时使用链表维护元素的次序,这使得元素看起来是以插入顺序保存的

Map

  1. Map与Collection并列存在,用于保存具有映射关系的数据:Key-Value
  2. Map中的key和value可以是任何引用类型的数据,会封装到HashMap$Node对象中
  3. Map中的key不允许重复,value可以重复
  4. Map的key可以为null,value也可以为null,但是key只能存在一个null,value可以存在多个为null
  5. 常用String类作为key
  6. key和value之间存在单向一对一关系,即指通过指定的key总能找到对应的value
